Creating A / B Test with Google Optimize

Review of Post 8

We have a goal for the event of a user clicking on the brand logo to get back to the home page of my user science journal. Moving forward, we will want to look at creating a variant of that brand logo to test against the original. This change doesn't need to be big; for our case, the difference will be using "USJ," instead of "Jon Hetrick."

Currently, with the limited traffic, I have had, the goal is not getting completed. People are visiting, but the data for the event is reflecting that users are not using the brand logo to navigate home. We already know that the brand logo is a usual way people navigate to the home page. Suggesting that we need to change what the brand logo is. By changing the brand to "USJ" (User Science Journal), I believe it will be a more obvious choice for the user to use to navigate home.

Test users interview

After talking with users informally, I further was able to back the statement of, users tend to default to the brand logo as there means of getting home. Users stated that they would click the top left logo area, even if they didn't know it was a button to try it. Furthermore, users reported that the naming doesn't need to be the name of the site, but it would make them more confident.

Setting up an Optimize A / B Test

Hypothesis:

"Changing the brand logo name of "Jon Hetrick" to "USJ" will decrease user's confusion and increase the number of clicks to take them to the home page."

The Process:

Following steps from HERE

  1. Log into Google Optimize
  2. Create a Container name and selected A / B Test
  3. Add the provided script tag to each page that you want to be able to benefit from Optimize.
  4. Create the A / B test on the optimized page.
  5. The change that you wish to test is all handled in the Optimize UI.
  6. Make sure to add all the URLs that will run this experience.
  7. Start the Test

Conclusion:

Overall the setup and creation of an A / B test with Google Optimize were painless and easy to setup. I like how the changes for the A / B test can be fone in the Optimize UI. No code changes were needed.
